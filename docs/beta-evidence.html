10 · Beta Evidence

Beta 指标证据

参与者在 Metrics 页人工复核并导出本机聚合值；没有后台上传，也不收集 Prompt、请求标识、日志或请求时间戳。

去重边界每个安装生成随机 128-bit 样本 ID，仅供离线聚合器拒绝重复贡献者。最终报告不保留样本 ID，只记录源文件 SHA-256。

门槛

Gate目标最小证据量
首次成功捕获 P50< 180,000 ms30 位贡献者、30 个合格样本
支持端点解析率> 95%30 位贡献者、1,000 个请求
Crash-free session> 99.5%30 位贡献者、1,000 个完成会话

权威配置为 release/beta-metrics-policy.v0.1.json。证据不足标记为 insufficient，不会伪装成通过或失败。

聚合

python scripts/aggregate_beta_metrics.py \
  --policy release/beta-metrics-policy.v0.1.json \
  --input-directory path/to/reviewed-exports \
  --output path/to/new-beta-report.json \
  --require-ready

工具拒绝重复样本、重复 JSON key、symlink、超限文件、错误版本、非目标平台、未知字段、不安全隐私声明、伪造率和违反子集关系的计数。输出必须是新文件。

审核

  1. 确认输入目录只含每位参与者最新且主动提交的一份导出。
  2. 保存聚合命令、策略版本、最终报告与受控原始证据位置。
  3. 由 Release Owner 审核三个 gate；未达标时继续取样或修复产品。
  4. 通过后仅将无样本 ID 的报告加入 readiness evidence；原始文件不进入仓库。

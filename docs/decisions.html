ADR-001 · 桌面技术栈

决策：Tauri 2 承载窗口、托盘与系统能力；React + TypeScript 构建 UI；Rust 管理可信边界。WebView 仅获得显式 capability。

替代：Electron 生态更成熟但运行时和攻击面更大；纯原生双端成本高；浏览器应用无法可靠管理本地代理和凭据库。

风险：WebView 平台差异、Tauri API 变动、原生签名复杂。通过跨平台 CI、最小权限和 command schema 控制。

复审：WebView 行为阻断核心检查器、Tauri 安全公告无法及时修复，或原生模块占比显著超过 UI 时。

ADR-002 · TLS 捕获 sidecar

决策：复用固定版本 mitmproxy/mitmdump；sidecar 不访问数据库，只向回环认证 IPC 发送已删除凭据的 envelope。

替代：自研 Rust MITM 风险和协议成本过高；系统注入不可维护；仅 Gateway 无法覆盖支持显式代理的第三方应用。

风险：Python 供应链、打包体积、进程崩溃与 CA 权限。通过有界队列、hash、签名、SBOM、watchdog 和最小环境变量控制。

复审：mitmproxy 无法满足签名/体积目标、关键协议长期不支持，或维护成本超过受控原生实现。

ADR-003 · 加密本地存储

决策：使用 SQLCipher、WAL 和版本化迁移；随机数据库 key 由 OS 凭据库持有，DB 文件、备份和 key 不共址。

替代：明文 SQLite + 字段加密易遗漏索引与临时页；RocksDB 缺少成熟关系查询与 FTS；整盘加密不覆盖文件复制场景。

风险：SQLCipher 跨平台构建、WAL/备份泄露、凭据库不可用与 key 丢失。实现必须包含锁定状态、恢复说明和 canary 扫描。

状态：实现与 Windows 本地凭据库实验已通过；等待 Windows/macOS CI 后转 Accepted。若 SQLCipher 发布或性能不可接受，评估页级加密替代。

ADR-004 · Prompt IR

决策：所有厂商适配器输出版本化 Prompt IR；每个 observed 节点保留原始 JSON 定位，derived/inferred 必须显式标记。

替代：直接展示厂商 JSON 无法比较；单一通用 chat schema 会丢失工具、缓存和多模态语义；仅采用 OTLP 无法表达完整证据。

风险：抽象泄漏与 schema 演进。通过 provider extensions、未知字段保留、fixture/golden 测试和版本迁移控制。

复审：连续两个主流协议无法无损表达，或版本迁移导致证据定位不可保持时。

ADR-005 · 捕获与代理策略

决策：优先 Gateway；其次应用显式代理；系统代理必须由用户主动启用，并在修改前持久化快照、启动独立 watchdog。证书固定失败时停止 MITM，不提供绕过。

替代：默认接管系统代理影响面过大；透明内核代理跨平台与权限成本高；浏览器扩展无法覆盖 CLI/IDE。

风险：代理残留、认证代理无法精确恢复、HTTP/3 和独立证书库不可见。通过拒绝不安全状态、能力诊断和明确降级路径控制。

复审：平台 API 变化破坏恢复保证、显式代理覆盖率不足，或浏览器/SDK 采集成为主要入口时。

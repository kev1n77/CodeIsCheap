Incident Response · 0.1

事故响应

优先恢复用户网络和保护本地数据；诊断只收集必要的事件代码、版本、平台和可验证制品信息。

立即升级代理残留导致断网、凭据/Prompt/CA 私钥泄露、签名异常、升级后主库与恢复备份均不可用。

分级

级别示例响应目标
P0大范围断网、发布制品或签名被替换、确认泄露私钥/凭据立即停止发布和分发；15 分钟内建立负责人
P1单设备代理无法恢复、升级后数据不可读、Prompt 进入非预期文件1 小时内缓解并保留最小证据
P2单应用兼容、解析错误、可回退的更新失败1 个工作日内给出路径
P3界面、文档或非阻断诊断问题进入常规版本计划

关键 Playbook

事件立即动作证据
系统代理残留停止 Proxy；运行应用启动恢复或同签名 watchdog；确认原 PAC/手动设置恢复平台、版本、恢复 journal 状态、事件代码、恢复前后设置
凭据或 Prompt 泄露停止分享/上传；删除副本；轮换凭据；暂停相关导出或发布命中文件类型、策略版本、最小复现；不得复制真实 secret
签名或制品异常保持 Release 为 draft 或下架；冻结 tag；核对 manifest、SHA-256、平台签名与 updater .sigActions run、release manifest、签名验证输出
升级后数据失败切回 Gateway；保持主库不变；只读打开受校验备份并导出必要历史源/目标版本、恢复 journal、备份字节数、兼容诊断代码

证据边界

  • 使用 支持包验证器生成摘要；不得把原始 Prompt、Raw、私钥或日志详情写入工单。
  • 发布事故保存 workflow URL、tag、manifest、文件哈希和签名结果。
  • 数据库问题先复制加密文件并限制访问；不得要求用户提供 OS 凭据库中的 key。
  • 证据到期按案件策略删除；安全事件只保留完成复盘所需的最小集合。

GA 演练

  1. 在隔离 Windows 与 macOS 设备分别模拟 sidecar 强杀、应用强杀和 owner 断连，确认网络恢复。
  2. 使用签名候选版本模拟迁移失败，确认只读历史、搜索和脱敏导出。
  3. 模拟 updater manifest 哈希或签名不匹配，确认不会公开或安装。
  4. 记录负责人、时间线、检测/缓解时间、证据链接和改进项；由非执行者复核。
  5. 用 verify_readiness_evidence.py init-manual --gate incident_response_drill 创建场景包；完成并校验后作为该 gate 的首项证据。

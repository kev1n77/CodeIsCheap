03 · Architecture

系统架构

本地优先架构：桌面 UI 负责检查体验，Rust Core 负责策略、解析和存储，成熟 sidecar 负责 TLS 网络流。

组件关系

技术选型

层选择理由
桌面端Tauri 2 + React + TypeScript跨平台桌面体验与原生系统能力
控制核心Rust + Tokio进程、网络、加密和性能
TLS 捕获mitmproxy sidecar复用成熟协议栈，避免自研 MITM
数据SQLite / SQLCipher本地事务、全文检索和加密
扩展声明式映射，复杂插件用 WASI最小权限、可隔离

架构原则

  • 网络转发优先于记录；压力过高时降级记录，不阻塞用户请求。
  • sidecar 不访问数据库和工作区密钥。
  • 新厂商适配器只产出 Prompt IR，不直接依赖 UI。
  • 未知协议和解析失败都可降级 Raw。
  • 系统代理由独立 watchdog 负责异常恢复。

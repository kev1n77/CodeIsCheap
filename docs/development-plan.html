07 · Development Plan

详细开发计划

以 20 周、10 个双周迭代完成桌面端 MVP。计划围绕“请求可捕获、Prompt 可解释、数据不泄露、网络可恢复”四条主线展开。

计划周期20 周
迭代长度2 周
目标平台Windows / macOS
计划基线2026-07-14
执行原则网络正确性与数据安全优先于功能数量。任何凭据落盘、系统代理无法恢复或推断冒充事实的问题都阻断发布。

5. 工程工作流

分支与评审

  • 主分支保持可发布。
  • 网络、安全、迁移和更新代码强制双人评审。
  • 每个适配器提交必须包含 fixture 与 golden 结果。
  • 所有行为变更更新对应设计文档或 ADR。

环境与制品

  • 开发环境只使用合成 API key 和本地 fake provider。
  • Canary、Beta、Stable 使用独立签名 channel。
  • 正式 sidecar 固定版本并记录 SBOM/hash。
  • 测试数据禁止包含真实用户 Prompt。

6. Definition of Ready / Done

Ready

  • 用户问题与验收结果明确
  • 数据与权限影响已标注
  • 依赖与 owner 已确认
  • 涉及协议时 fixture 已准备
  • 涉及风险时有回退方案

Done

  • 代码、测试和文档已评审
  • 错误、空、加载和降级状态完整
  • 日志与持久化通过 secret 扫描
  • Windows/macOS 对应测试通过
  • 指标可观测且回滚路径验证

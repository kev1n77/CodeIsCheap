06 · Delivery

MVP 交付

先证明“稳定看到并正确解释请求”，再扩展应用数量、团队协作和 AI 分析。

范围切线

MVP 必须有

  • Gateway + 显式代理
  • 四类协议适配
  • Anatomy / Raw / Timeline
  • 结构化比较
  • 本地搜索与脱敏导出
  • Windows / macOS

后续再做

  • 浏览器扩展与 TUN
  • 团队云同步
  • AI 语义分析
  • 完整评测平台
  • 移动和远程设备
  • Linux GA

建议阶段

Discovery · 2 周协议样本、Gateway/代理 Spike、威胁模型、Prompt IR。
Foundation · 3 周桌面框架、策略管线、加密存储、适配器注册表。
Private Alpha · 6 周Gateway、OpenAI/Anthropic、代理恢复、Inspector、导出。
Public Beta · 5 周Gemini/Ollama、Compare、规则、性能和安全评审。
GA · 4 周签名更新、升级回滚、文档、本地化和发布演练。

发布门槛

  • 已支持端点解析成功率高于 95%。
  • Windows 与 macOS 的代理恢复故障测试全部通过。
  • 数据库、日志、临时文件和导出物中无鉴权凭据。
  • 所有推断字段都明确标记，不伪装成 observed。
  • 未知协议可降级 Raw，sidecar 崩溃不影响历史数据。

建议团队

核心 6 人：1 名技术负责人、1 名网络/桌面工程师、1–2 名前端工程师、1 名设计师、1 名 QA/SDET，安全评审按阶段参与。

最高风险

风险应对
应用绕过代理或证书固定多采集模式，不宣传 universal capture
凭据或 Prompt 泄露最小范围、双层 scrub、加密与强制预览
崩溃导致断网独立 watchdog；恢复率不达标则不发布系统代理模式
范围膨胀把“理解真实请求”作为唯一首发闭环
